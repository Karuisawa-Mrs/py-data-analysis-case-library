# 案例概述

本案例构造一个随机营销触达场景：企业可以向部分客户发送优惠触达，但并非所有“高响应客户”都值得被投放。真正有业务价值的是 **uplift 高** 的客户，即“因为触达才更可能转化”的客户，而不是本来就会下单的自然高响应人群。

案例使用模拟客户样本演示双模型（two-model / T-learner 风格）uplift 建模：分别拟合处理组与对照组的响应概率，再用两者差值近似个体增量响应。输出包括 uplift 排名表、不同投放策略下的 ROI 对比，以及顶部客户名单。

## 分析设计

- `analysis.py` 生成 5000+ 随机实验风格的客户数据，包含处理、响应、基线概率与真实增量效应。
- 处理组与对照组分别训练二元响应模型，形成 `mu1_hat - mu0_hat` 的 uplift 估计。
- 同时保留 `mu1_hat` 作为“预测响应”分数，以便与 uplift 排序做 ROI 对比。
- ROI 评估比较三种策略：按 uplift 排序投放、按预测响应排序投放、随机投放。

## 复现说明

```bash
python analysis.py --smoke-test
python analysis.py
```

## 结果说明

- `outputs/uplift_ranking.csv`：按 uplift 分数分位汇总的客户排序结果。
- `outputs/roi_comparison.csv`：不同预算比例与投放策略下的预期增量转化与 ROI 对比。
- `outputs/top_customers.csv`：最值得优先触达的客户样本。
- `outputs/summary.txt`：核心结论、GitHub 适配说明与边界声明。
