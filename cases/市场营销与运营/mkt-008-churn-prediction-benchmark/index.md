# 案例概述

客户流失预测是营销分析与客户经营中的典型二元分类任务。实际业务里，流失客户往往占比不高，若只看准确率，很容易掩盖模型对少数高风险客户识别不足的问题。因此，除了比较模型本身，还需要显式处理类别不平衡，并同时关注 ROC AUC、平均精确率（Average Precision）、PR 曲线与阈值下的混淆矩阵。

本案例构造一个模拟的电信订阅客户数据集，对比 **Logistic Regression** 与 **Gradient Boosting** 两类常见预测模型。逻辑回归便于解释线性方向与边际关系；梯度提升树更擅长拟合非线性与特征交互。案例重点不是宣布“谁在真实世界一定更好”，而是演示一个规范的预测 benchmark 流程。

## 分析设计

- 样本：模拟电信订阅客户，包含套餐、账单、服务质量、满意度与客服互动等特征。
- 任务：预测客户是否在观察窗口内流失（`churned`）。
- 方法：统一训练/测试切分，对逻辑回归使用 `class_weight="balanced"`，对梯度提升树使用平衡样本权重。
- 评估：输出 ROC AUC、Average Precision、PR 曲线、基于 F1 阈值的混淆矩阵，以及逻辑回归系数与梯度提升树特征重要性。
- 解释边界：数据生成机制由脚本预先设定，因此所有结果仅用于演示预测建模逻辑，不能推出真实电信业务中的经验规律或因果关系。

## 结果说明

- `outputs/model_metrics.csv`：两种模型在 holdout 集上的 ROC AUC、Average Precision、平衡准确率、F1 与最优阈值。
- `outputs/precision_recall_curve.png`：两种模型的 PR 曲线对比，适合在流失比例偏低时比较识别能力。
- `outputs/confusion_matrices.png`：按各自 F1 最优阈值生成的混淆矩阵，便于观察查全率与误报之间的权衡。
- `outputs/feature_importance_comparison.csv`：逻辑回归绝对系数与梯度提升树特征重要性对比。
- `outputs/summary.txt`：记录模拟设定、样本流失率、模型比较结论与解释边界。
